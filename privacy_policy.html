Privacy Policy – Moodle Bulk Downloader

Last updated: April 2026

Moodle Bulk Downloader ("the extension") is a browser extension for Microsoft Edge and Google Chrome that lets users download course materials from a Moodle learning-management-system page in bulk. This policy explains what data the extension accesses and how it is handled.

1. Data collected

The extension does not collect, transmit, or store any personal data on remote servers. All processing happens locally inside your browser.

2. Data accessed locally

  • Active tab URL and title – read once when you open the extension popup, used only to construct a default filename for ZIP downloads and to identify which page to scan.
  • Page content (DOM) – a content script reads anchor tags on the active Moodle course page to discover downloadable file links. No page content is sent anywhere.
  • HTTP response headers – the extension's background service worker makes HEAD/GET requests to the file URLs found on the page (using your existing Moodle session cookies) to resolve accurate filenames from Content-Disposition headers. These requests go directly to your Moodle server. No data passes through any third-party server.
  • Downloaded file bytes – when using ZIP mode the extension fetches file bytes into memory to build the ZIP archive. The bytes are discarded from memory as soon as the ZIP is saved to disk.

3. Permissions used

  • activeTab – to read the URL and inject the page-scanning script into the current tab.
  • scripting – to programmatically inject the content script only when you open the popup.
  • downloads – to save files to your browser's default Downloads folder.
  • storage – to persist a single cancellation flag in chrome.storage.session (cleared when the browser session ends).
  • Host permissions (optional) – requested once per Moodle domain to allow the background service worker to fetch files with your session credentials. You are prompted by the browser before this permission is granted.

4. Third-party services

The extension bundles JSZip (MIT licence) locally. No CDN or external script is loaded at runtime. No analytics, telemetry, or advertising libraries are included.

5. Data sharing

No data is shared with the extension developer or any third party. All network requests are made directly between your browser and your institution's Moodle server.

6. Data retention

The extension stores nothing beyond the current browser session. Closing the browser clears any session storage used by the extension.

7. Contact

For questions about this policy please open an issue on the GitHub repository.
